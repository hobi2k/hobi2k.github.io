# Wan 2.2 SVI 영상 생성 세팅 가이드

이 글은 **WAN 2.2 + SVI 워크플로우를 처음부터 다시 세팅하는 과정**만 깔끔하게 정리한 문서다.

핵심 목표는 하나다.

- 기존 ComfyUI 환경에서 버전 충돌, SageAttention, Triton, 커스텀 노드 누락 때문에 막히지 않고
- **새 폴더에서 WAN 2.2 SVI 워크플로우를 바로 실행 가능한 상태**까지 가는 것

기존 ComfyUI를 오래 굴리면서 환경이 꼬였다면, 모델 폴더만 백업해두고 새로 세팅하는 쪽이 더 빠를 수 있다.


## 1. 어떤 상황에서 이 세팅이 필요한가

다음 같은 상황이면 이 방식이 잘 맞는다.

- Wan 2.2를 처음 만지는 경우
- 다른 글 따라하다가 Torch / CUDA / Triton / SageAttention에서 계속 막히는 경우
- ComfyUI는 되는데 SVI 워크플로우만 안 도는 경우
- 커스텀 노드 누락 때문에 워크플로우 import 후 에러가 뜨는 경우

이 가이드는 **Windows + ComfyUI + Wan 2.2 + SVI** 조합을 기준으로 적는다.


## 2. 준비 환경

원문 작성자 기준 환경은 다음과 같다.

- GPU: RTX 5080
- VRAM: 16GB
- RAM: 64GB
- NVIDIA 드라이버: 591.59
- Python: 3.11.9
- CUDA: 13.1
- Git 설치

실사용 관점에서는 아래 정도로 이해하면 된다.

- Python 3.11 계열
- CUDA 13 계열
- 최신 NVIDIA 드라이버
- Git 설치 완료
- VRAM 16GB 전후면 14B 계열 워크플로우 접근 가능


## 3. 필요한 모델과 파일 위치

워크플로우 기준으로 필요한 파일은 아래와 같다.

### diffusion_models

- Smooth Mix Wan 2.2 (I2V/T2V 14B)
  - 위치: `ComfyUI/models/diffusion_models`
  - 링크: https://civitai.com/models/1995784?modelVersionId=2260110

- DaSiWa WAN 2.2 I2V 14B
  - 위치: `ComfyUI/models/diffusion_models`
  - 링크: https://civitai.com/models/1981116?modelVersionId=2388627

### loras

- `SVI_v2_PRO_Wan2.2-I2V-A14B_HIGH_lora_rank_128_fp16.safetensors`
- `SVI_v2_PRO_Wan2.2-I2V-A14B_LOW_lora_rank_128_fp16.safetensors`
  - 위치: `ComfyUI/models/loras`
  - 링크: https://huggingface.co/Kijai/WanVideo_comfy/tree/main/LoRAs/Stable-Video-Infinity/v2.0

### clip

- `umt5_xxl_fp16.safetensors`
  - 위치: `ComfyUI/models/clip`
  - 링크: https://huggingface.co/Comfy-Org/Wan_2.1_ComfyUI_repackaged/blob/main/split_files/text_encoders/umt5_xxl_fp16.safetensors

### vae

- `wan_2.1_vae.safetensors`
  - 위치: `ComfyUI/models/vae`
  - 링크: https://huggingface.co/Comfy-Org/Wan_2.1_ComfyUI_repackaged/blob/main/split_files/vae/wan_2.1_vae.safetensors


## 4. 폴더 생성과 가상환경 준비

예시는 `C:\AI\ComfyUI` 기준이다.

1. `C:\AI\ComfyUI` 폴더를 만든다.
2. 폴더에서 `Shift + 우클릭` 후 PowerShell을 연다.
3. 아래 명령으로 가상환경을 만든다.

```powershell
python -m venv venv
venv\Scripts\activate
```

정상적으로 되면 명령 프롬프트 앞에 `(venv)`가 붙는다.


## 5. Torch와 CUDA 조합 맞추기

원문 기준 핵심 조합은 아래다.

```powershell
pip install --upgrade pip
pip install torch==2.10.0 torchvision torchaudio --index-url https://download.pytorch.org/whl/cu130
```

설치 후 확인:

```powershell
python -c "import torch; print(torch.__version__, torch.version.cuda)"
```

여기서 버전이 어긋나면 뒤에서 SageAttention과 Triton에서 다시 꼬일 수 있다.


## 6. ComfyUI 설치

```powershell
git clone https://github.com/comfyanonymous/ComfyUI.git
cd ComfyUI
pip install -r requirements.txt
```

그 다음 Triton 설치:

```powershell
pip install triton
```


## 7. SageAttention 설치

원문은 Windows용 사전 빌드 wheel을 받아서 설치하는 방식이다.

다운로드:

- `sageattention-2.2.0+cu130torch2.9.1-cp311-cp311-win_amd64.whl`
- 링크: https://github.com/sdbds/SageAttention-for-windows/releases/download/torch291%2Bcu130/sageattention-2.2.0+cu130torch2.9.1-cp311-cp311-win_amd64.whl

이 파일을 `ComfyUI` 바깥 한 단계 위 폴더, 즉 예시 기준 `C:\AI\ComfyUI` 쪽에 둔다.

그 뒤 다시 활성화해서 설치:

```powershell
venv\Scripts\activate
pip install ..\sageattention-2.2.0+cu130torch2.9.1-cp311-cp311-win_amd64.whl
```

**혹은 ComfyUI 삽질 기록3 참조**

## 8. 실행용 run.bat 만들기

원문은 실행을 반복하기 편하게 `run.bat`를 만드는 방식을 권장한다.

`C:\AI\ComfyUI`에 `run.bat`를 만들고 아래 내용을 넣는다.

```bat
@echo off
cd /d %~dp0
call venv\Scripts\activate
echo [INFO] Starting ComfyUI server...
start "" cmd /c "python main.py --force-fp16"
echo [INFO] Waiting for ComfyUI (port 8188)...
powershell -Command ^
"while (-not (Test-NetConnection -ComputerName 127.0.0.1 -Port 8188 -InformationLevel Quiet)) { Start-Sleep -Milliseconds 500 }"
echo [INFO] ComfyUI is ready. Opening browser...
start http://127.0.0.1:8188
pause
```

이렇게 해두면 앞으로는 `run.bat` 더블클릭만으로 실행 가능하다.

원문처럼:

- 바탕화면 바로가기 생성
- 작업표시줄 고정

까지 연결해도 된다.


## 9. ComfyUI Manager 확인

ComfyUI 실행 후 `Manager`가 보이는지 확인한다.

없다면:

```powershell
venv\Scripts\activate
cd custom_nodes
git clone https://github.com/ltdrdata/ComfyUI-Manager comfyui-manager
```

그 뒤 ComfyUI를 재시작한다.

Manager가 있다면:

1. `Manager` 클릭
2. `Install Missing Custom Nodes` 클릭
3. 뜨는 항목 전부 설치
4. ComfyUI 재시작

## 12. 처음 돌릴 때 같이 기억할 점

원문과 댓글에서 특히 중요한 포인트만 추리면 아래다.

### 1) xformers는 설치하지 않는다

원문에서 가장 강하게 적어둔 주의사항 중 하나다.

- `xformers를 설치하지 마세요`

환경 꼬임을 줄이려면 괜히 더 깔지 않는 편이 낫다.

### 2) SVI는 일관성 유지용이고, WAN이 실제 영상 모델이다

댓글 흐름을 보면 초보자가 가장 많이 헷갈리는 부분이 이거다.

- WAN: 실제 영상 생성 모델
- SVI: 짧은 영상 클립을 더 안정적으로 연결하고 일관성을 유지하는 쪽에 가까운 워크플로우

즉 “WAN 없이 SVI만”이 아니라, **WAN 2.2 기반 위에서 SVI 워크플로우를 얹는 구조**로 이해하면 된다.

### 3) 모델 선택 팁

댓글 기준으로 언급된 팁은 다음과 같다.

- Smooth Mix는 링크의 `high i2v`부터 보기
- DaSiWa는 링크 쪽 `low`부터 시작해보기

### 4) 브라우저 에러는 하단 메시지를 복사해서 먼저 확인

원문도 같은 얘기를 한다.

- 브라우저에서 에러가 뜨면 제일 아래까지 내려서 내용 확인
- 오류 메시지를 복사해서 정리
- 그 텍스트를 기준으로 원인 파악

실제로 많이 걸리는 쪽은:

- Python 버전 mismatch
- Triton import 문제
- SageAttention wheel 버전 mismatch
- 커스텀 노드 누락
- 프론트엔드 노드 에러


## 13. 이 가이드가 유용한 이유

이 세팅의 장점은 “이론 설명”보다 **처음부터 끝까지 한 번에 맞는 조합으로 고정한다**는 점이다.

특히 아래 같은 사람에게 잘 맞는다.

- Wan 2.2를 처음 세팅하는 사람
- ComfyUI는 알지만 영상 쪽은 처음인 사람
- 이것저것 섞다가 환경이 꼬인 사람
- SVI 워크플로우만 빨리 재현해보고 싶은 사람


## 14. 정리

한 줄 요약하면 이렇다.

- ComfyUI를 새 폴더에 설치하고
- Python 3.11 + Torch 2.9.1 cu130 + Triton + SageAttention 조합을 맞춘 뒤
- 필요한 WAN 2.2 모델과 SVI LoRA를 정확한 폴더에 넣고
- `svi.pro_workflow_mmmmn.json`을 불러오면 된다

중간에 가장 많이 막히는 건 대개 아래 둘이다.

- 버전 mismatch
- 누락된 커스텀 노드

그래서 기존 환경을 계속 억지로 고치는 것보다, **처음부터 이 워크플로우 기준으로 새로 세팅하는 방식**이 훨씬 수월할 수 있다.
