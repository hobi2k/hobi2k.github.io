# 12월 5일 협업일지

날짜: 2024-12-05
이름: 안호성
팀명: 4팀

## 오늘 맡은 역할 및 작업 내용

프로젝트 초기 환경 구축 및 데이터 파이프라인 골격 설계.

- 기본적인 협업 절차 구축
- 깃허브 리포지토리 생성 및 기본 README 초안 작성
- 프로젝트 폴더 구조 설계 (data/, src/, detection/, utils/ 등 베이스 레이아웃 수립)
- Config 클래스 초안 구현 (데이터 경로 및 주요 상수 정의)
- Kaggle 알약 인식 데이터셋 로컬/서버 환경으로 로드
- 데이터 alignment 파이프라인 구현 (annotation 정합성 확인, 이미지–라벨 매칭 확인)

## 오늘 작업 현황

프로젝트 베이스 세팅과 데이터 정렬까지 완료하였으나,
시각화(EDA) 코드는 작성 중 시간이 부족하여 다음 작업일(월요일)로 이월했다.

## 오늘 협업 중 제안하거나 피드백한 내용

- 프로젝트 초기부터 역할을 과도하게 분업하지 말자는 의견을 제안했다.
    - 처음부터 분업하면 가각 특정 기능만 담당하게 되어 개별 스킬 편향이 생길 수 있다고 생각했다.
    - 전문화된 영역은 일반적인 흐름을 체험한 뒤 충분히 전개해 볼 수 있으리라 생각했다.
- 결론적으로 기본 데이터 파이프라인은 팀원 전원이 한 번씩 직접 구성해보고 이해한 뒤, 이후 단계에서 분업하자는 방향성을 제안헸디.

## 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점

- Kaggle의 원시 데이터셋에서 annotation은 기본적으로 COCO JSON 형식을 취하고 있었는데, 디렉토리 구조가 너무 복잡하여 alignment 작업과 시각화를 진행할 필요성이 큼을 확인했다.
- annotation 일부가 누락되거나 매칭되지 않는 사례가 있어 이후 자동 검증 루틴이 필요하다는 점을 발견했다.
- Config 기반 프로젝트 관리가 기능 확장 시 유지보수에 큰 장점으로 작용할 것임을 이미 알고 있었고, 이를 기반으로 코드를 작성했다.

## 일정 지연이나 협업 중 어려웠던 점

- 팀장님의 부재로 대신 프로젝트 방향성을 주도했다.
- 데이터셋 구조를 파악하는 데 많은 시간이 필요할 것임을 직감했다.

## 오늘 발표 준비나 커뮤니케이션에서 기여한 부분

- 프로젝트 시작 방향성 명확화(전체 흐름 공통 학습 -> 이후 분업 구조).
- 리포지토리 기본 구조 및 README 초안 공유하여 팀 내 공통 이해 기반 마련.
