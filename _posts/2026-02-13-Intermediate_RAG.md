# 2월 13일 협업일지

날짜: 2026-02-13  
이름: 안호성  
팀명: 3팀

## 오늘 맡은 역할 및 작업 내용

오늘은 발표 리허설을 진행한 후, 중간발표를 실제로 수행했다.  
발표 종료 후 받은 피드백을 바탕으로 발표자료를 전면 재구성하기로 결정했다.

1. 발표 리허설 및 중간발표 진행

- 전체 발표 흐름 최종 점검
  - 시간 초과 구간 조정
  - 기술 설명 간 연결 멘트 보완
- 중간발표 수행
  - 시스템 전체 구조 설명
  - 데이터 전처리, RAG 파이프라인, TTS 구조, SQL 저장 흐름 발표
  - 실제 구현 기반 설명에 집중

2. 발표 피드백 정리

- 메시지가 다소 분산되어 있다는 의견 확인
- "핵심 문제 -> 해결 구조 -> 결과" 흐름을 더 강화할 필요성 제기
- 기술 구현보다 문제 정의와 차별점이 더 부각되어야 한다는 피드백 수렴

3. 발표자료 재구성 작업

- 기존 슬라이드를 유지하는 방식이 아니라 구조 재설계를 선택
- manus를 활용해 전체 발표자료 구조 재정렬
- 문제 정의 -> 시스템 아키텍처 -> 데이터 흐름 -> 기술 차별점 중심으로 재배치
- 재구성한 자료를 구글슬라이드로 변환 후 팀원들과 공유
- 실시간 공동 수정이 가능하도록 협업 환경 구축

## 오늘 작업 현황

- 중간발표 완료
- 피드백 정리 완료
- 발표자료 2차 구조 재설계 완료
- 팀원들과 실시간 수정 가능한 슬라이드 공유 완료

## 오늘 협업 중 제안하거나 피드백한 내용

- 발표는 기능 설명이 아니라 "해결 전략" 중심으로 가야 한다고 제안
- 공동 수정이 가능한 환경에서 빠르게 재정리하자고 제안

## 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점

- 기술 구현이 잘 되어 있어도 메시지 구조가 약하면 설득력이 떨어짐
- 발표는 아키텍처 설명보다 문제 해결 흐름을 보여주는 것이 중요함
- 협업 문서는 실시간 수정 환경이 확보되어야 속도가 붙음

## 일정 지연이나 협업 중 어려웠던 점

- 발표 직후 피로도가 높은 상태에서 구조 재설계를 진행해야 했음
- 슬라이드 전체를 다시 설계하는 데 시간이 소요됨

## 오늘 발표 준비나 커뮤니케이션에서 기여한 부분
