# 12월 8일 협업일지

날짜: 2024-12-08
이름: 안호성
팀명: 4팀

## 오늘 맡은 역할 및 작업 내용

데이터 시각화 파트 완성 및 YOLO 모델 적용을 위한 전처리 파이프라인의 본격적인 구축을 수행했다.

- 이미지–라벨 정합성 검증용 시각화 코드 완성
    - annotation이 존재하는 파일만 필터링
    - 이미지 위에 bounding box를 그려 결과물을 저장하는 파이프라인 설계 및 구현
- EDA 결과 분석
    - 전체 651개 이미지 중 annotation이 존재하는 유효 이미지가 232개뿐임을 확인
    - 일부 annotation 파일에서도 박스 누락/좌표 튐 문제 존재
    - 하지만 이번 주차 목표는 “작동 가능한 전체 파이프라인 구축”이므로 정교한 클린업은 후순위로 미루기로 결정
- YOLO 모델 검토
    - 학습에 사용할 모델 버전 조사 (YOLOv8, YOLOv9, YOLOv11 비교)
    - 초기 버전으로는 가볍고 빠른 YOLOv8n을 사용하기로 내부 결정
- YOLO 학습을 위한 데이터셋 구축 착수
    - COCO annotation -> YOLO 포맷 변환기 개발
    - COCO parser 코드 작성
- 나는 매일 작성한 코드를 GitHub에 커밋 및 푸시하여 리포지토리의 변화를 지속적으로 관리하고 내 코드를 전체에 투명하게 공유했다.

## 오늘 작업 현황

시각화 파트를 마무리하고 YOLO 모델 선정 및 COCO 파서 개발도 상당 부분 완료했다.
다만 YOLO 학습을 위한 전체 데이터셋 전처리 파이프라인 구축은 미완성 단계로 남았다.

## 오늘 협업 중 제안하거나 피드백한 내용

- annotation 품질 이슈(박스 누락, 잘못된 좌표 등)는 현재 단계에서는 해결하지 않고, 우선 전체 파이프라인 완성을 목표로 하자는 의견을 제시했다.
- YOLO 모델 선택 과정에서 YOLOv8n의 장점을 설명했다.
    - 빠른 실험 가능
    - 부족한 데이터 상황에서도 baseline 확보 용이

## 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점

- 전체 데이터(651개) 중 실제로 annotation이 존재하는 샘플 수(232개)가 생각보다 많이 적다는 점을 인지했다.
    - 데이터 증강, 라벨링, 외부 데이터 확보 전략을 장기적으로 고려해야 한다.
- annotation 파일이 있어도 박스가 제대로 매칭되지 않는 사례가 간헌적으로 존재함을 발견했다.
    - 향후 “annotation 자동 검증기” 또는 “pseudo labeling 기반 보정 파이프라인”의 개발이 필요함을 인지했다.
- YOLO 모델 선정 과정에서, 체급이 작은 모델을 먼저 실험하는 것이 프로젝트 초기 속도를 유지하는 데 가장 중요하다는 점을 확인했다.

## 일정 지연이나 협업 중 어려웠던 점

- 데이터셋 전체의 annotation 품질이 균일하지 않아 검증 작업에 시간이 소요되었다.
- 팀원 대부분이 아직 GitHub 활용이 미숙하여 리포지토리 관리가 나에게 집중되는 문제가 있다.
- YOLO용 데이터 포맷(YOLO TXT 방식)과 COCO(JSON) 매핑 규칙을 파악하는 데 예상보다 많은 시간이 필요했다.
