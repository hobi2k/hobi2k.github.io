# 2월 12일 협업일지

날짜: 2026-02-12  
이름: 안호성  
팀명: 3팀

## 오늘 맡은 역할 및 작업 내용

오늘은 중간 발표 직전 최종 정리 단계로,  
팀원들과 전체 발표자료를 완성했고 나는 발표용 대본을 작성했다.  

슬라이드에 맞춰 기술 설명 흐름을 정리하고,  
시연 구간에서 끊기지 않도록 발표 동선을 구성하는 데 집중했다.

1. 발표 대본 작성

- 전체 발표 흐름 정리
  - 문제 정의 -> 시스템 구조 -> 파싱, 임베딩, 리트리버 테스트 -> RAG 파이프라인 -> TTS -> DB 저장 구조 순으로 재구성
- 기술 설명 간 연결 멘트 작성
- TTS 설명 대본 작성
  - ONNX 기반 합성 구조 요약
  - 비동기 워커 처리 방식의 필요성 설명
  - 실제 서비스 환경에서의 안정성 고려 포인트 정리
- SQL 데이터베이스 설명 대본 작성
  - 대화 저장 스키마 구조 설명
  - 질문 -> 답변 -> rating 저장 흐름 서술
  - 저장 데이터의 활용 목적(피드백 기반 개선) 강조

2. 발표 흐름 점검

- 슬라이드 전환 타이밍에 맞춘 멘트 수정

## 오늘 작업 현황

- 발표용 전체 대본 작성 완료
- 시연 흐름에 맞춘 멘트 수정 완료

## 오늘 협업 중 제안하거나 피드백한 내용

- 시연에서는 음성과 DB 출력도 보여주어야 한다고 제안

## 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점

- 슬라이드만 보면 구현 깊이가 충분히 드러나지 않을 수 있음
- 발표 대본이 구조를 보완해주는 역할을 해야 함

## 일정 지연이나 협업 중 어려웠던 점

- 발표 분량을 줄이면서도 기술적 깊이를 유지하는 것이 어려웠음
- 시간 제한 내에서 모든 파트를 균형 있게 배치하는 데 고민이 필요했음

## 오늘 발표 준비나 커뮤니케이션에서 기여한 부분

- 전체 발표 대본 작성
- TTS 및 SQL 구조 설명 멘트 구체화
- 발표 흐름 자연스럽게 연결되도록 구조 보완

## 내일 목표 / 할 일

- 중간 발표 진행
