# 12월 19일 협업일지

날짜: 2024-12-19
이름: 안호성
팀명: 4팀

## 오늘 맡은 역할 및 작업 내용

오늘은 그동안 구현해 온 Stable Diffusion + SAM 기반 이미지 증강 파이프라인을
실제로 실행, 시연 가능한 데모 형태로 완성하고, 비개발자나 윈도우 환경에서도 쉽게 실행할 수 있도록
**Windows용 실행 배치 파일(.bat)**을 구현했다.

1. Stable Diffusion + SAM 증강 시스템 데모 구현

- 기존 상태
    - Diffusion + SAM 증강 파이프라인은 코드 레벨에서 완성
    - 다만 개발자 기준 실행에 최적화되어 있어 접근성이 낮았음

- 오늘 작업
    - 증강 파이프라인을 단일 실행 흐름으로 정리
    - 입력 이미지 -> SAM mask 생성 -> background inpainting -> 결과 저장 과정이 한 번에 동작하도록 데모 스크립트 구성
- 데모 목표
    - “이론적으로 가능한 증강”이 아니라
    - 실제로 클릭/실행해서 결과를 확인할 수 있는 증강 시스템 제공
- 결과
    - 알약은 그대로 유지된 상태에서
    - 배경만 자연스럽게 변경된 증강 이미지 생성 확인

2. Windows용 실행 배치 파일(.bat) 구현

- 문제 인식
    - 팀원 다수가 Windows 환경 사용
    - Python 스크립트 직접 실행은 진입 장벽이 큼
- 해결
    - 증강 데모를 .bat 파일로 실행 가능하도록 구성
    - 배치 파일에서:
        - 파이썬 확인 및 설치
        - 가상환경 활성화
        - 필요한 스크립트 호출
        - 실행 로그 출력 이 한 번에 이루어지도록 구현
- 효과
    - Python 환경에 익숙하지 않은 사용자도 더블클릭만으로 증강 데모 실행 가능
    - 시연, 발표, 검증 단계에서 활용도 크게 증가

3. 증강 시스템 사용 흐름 정리

- 증강 시스템이 다음 조건을 만족함을 재확인
    - 알약 객체 자체는 절대 변경되지 않음
    - YOLO annotation 재사용 가능
    - 증강 수, 프롬프트, 네거티브 프롬프트는 config로 제어
- 결과적으로
    - 연구용 코드 -> 실사용 가능한 증강 도구로 전환 완료

## 오늘 작업 현황

오늘은 기능 구현보다 “사용 가능성”을 끌어올린 날이었다.

- Diffusion + SAM 증강은 더 이상 실험 코드가 아니라 실제로 돌려볼 수 있는 데모 시스템
- Windows용 실행 파일 도입으로 팀 내부 공유, 발표 시연이 훨씬 수월해졌다.

## 오늘 협업 중 제안하거나 피드백한 내용

- 팀원들에게 증강 시스템은 코드 설명보다 직접 실행해보는 것이 가장 빠른 이해 방법이라는 점을 공유

## 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점

- 좋은 파이프라인도 실행이 어려우면 협업에서 활용되지 않음
- 배치 파일 등 “접근성 개선 작업”은 기술 난이도는 낮아 보여도 프로젝트 완성도를 크게 끌어올림

## 일정 지연이나 협업 중 어려웠던 점

- Windows 환경별 Python/가상환경 차이로 배치 파일 테스트에 시간이 소요됨

## 오늘 발표 준비나 커뮤니케이션에서 기여한 부분

- Stable Diffusion + SAM 증강 데모를 팀원에게 공유
- bat 파일 실행 방식 설명을 통해 팀원들이 직접 증강 결과를 확인할 수 있도록 지원함
