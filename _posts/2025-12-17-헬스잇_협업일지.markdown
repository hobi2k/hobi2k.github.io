# 12월 17일 협업일지

날짜: 2024-12-17
이름: 안호성
팀명: 4팀

## 오늘 맡은 역할 및 작업 내용

오늘은 개발 단계에서 통합 단계로 넘어간 날이었다.
Dev 브랜치에 병합된 팀원 코드에 맞추어 Stable Diffusion 기반 이미지 증강 파이프라인을 리팩토링했고,
프로젝트 전반을 관통하는 **멀티모달 알약 인식 시스템(YOLO + Qwen + SpeechT5)**을
Gradio 기반 데모 애플리케이션으로 구현했다.

1. Stable Diffusion 이미지 증강 파이프라인 리팩토링

- Dev 브랜치에 통합된 팀원 코드 구조를 기준으로 기존에 단독으로 개발하던 Diffusion + SAM 증강 코드 전면 리팩토링
- 수행 내용
    - 디렉토리 구조 Dev 브랜치 기준으로 정렬
    - 공용 config / utils 구조에 맞게 코드 수정
    - 증강 파이프라인 진입점 정리
- 목표
    - 개인 실험 코드 -> 팀 공용 파이프라인 코드로 전환
    - Dev 브랜치 기준에서 누구나 실행 가능한 상태 확보

2. Stable Diffusion 증강 파이프라인 문서화

- 리팩토링된 증강 파이프라인을 기준으로 문서 작성
- 문서화 범위
    - 파이프라인 전체 흐름 설명
    - SAM mask 기반 배경 inpainting 원리
    - aug_config를 통한 제어 방법
    - 사용 시 주의사항 (알약 왜곡 방지 원칙)
- 목적
    - 팀원들이 코드를 보지 않아도 구조를 이해할 수 있도록 정리
    - 이후 증강 전략 변경 시 기준 문서로 활용 가능하게 함

3. 멀티모달 알약 인식 시스템 구현 (YOLO + Qwen + SpeechT5)

- 기존에 개별적으로 존재하던 모델들을 하나의 흐름으로 통합
- 구성
    - YOLO: 이미지에서 알약 객체 탐지
    - Qwen LLM: 탐지 결과를 바탕으로 텍스트 설명 생성
    - SpeechT5: Qwen이 생성한 설명을 음성으로 출력

- 설계 포인트
    - 각 모델을 독립적으로 호출하되
    - 입력/출력 포맷을 명확히 정의하여 느슨하게 결합
    - 이후 모델 교체가 가능하도록 구조 설계

4. Gradio 기반 통합 데모 구현

- Gradio를 사용해 멀티모달 시스템을 하나의 UI로 통합
- 기능
    - 이미지 업로드
    - YOLO 기반 알약 탐지 결과 시각화
    - 텍스트 설명 출력
    - 음성(TTS) 출력
- 의미
    - 단순 모델 실험을 넘어서
    - 실제 사용 가능한 알약 인식 애플리케이션 형태로 구현
    - 팀원 및 외부 사용자에게 시스템을 직관적으로 설명 가능

## 오늘 작업 현황

오늘은 프로젝트가 **개별 기술 실험 단계 -> 통합된 시스템 단계**로 넘어간 날이다.
- 증강 파이프라인은 팀 코드 기준으로 정리되었고
- 멀티모달 알약 인식 시스템이 실제 동작하는 데모 형태로 구현되었다.

## 오늘 협업 중 제안하거나 피드백한 내용

- 증강 파이프라인은 개인 코드가 아닌 Dev 브랜치 기준 구조를 우선으로 유지해야 한다는 점을 공유
- 멀티모달 시스템은
    - 연구용이 아니라
    - “사용자에게 보여줄 수 있는 형태”가 중요하다는 의견 제시
- Gradio 데모를 기준으로 향후 발표 및 시연 방향을 잡을 수 있음을 설명

## 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점

- 기능이 늘어날수록 코드 품질보다 구조 정합성이 더 중요해짐
- 멀티모달 시스템에서는 각 모델의 성능보다 모델 간 인터페이스 설계가 전체 안정성을 좌우함
- Gradio UI는 디버깅, 커뮤니케이션, 데모 세 역할을 동시에 수행할 수 있는 강력한 도구임을 재확인

## 일정 지연이나 협업 중 어려웠던 점

- Dev 브랜치 코드와 개인 코드 간 구조 차이로 리팩토링에 예상보다 많은 시간 소요
- 여러 모델(YOLO, Qwen, SpeechT5)을 동시에 다루면서 환경 설정 및 리소스 관리에 신경을 많이 써야 했음

## 오늘 발표 준비나 커뮤니케이션에서 기여한 부분

- Gradio 기반 멀티모달 데모를 통해 팀원들이 프로젝트의 “최종 그림”을 직관적으로 이해할 수 있도록 기여
- 증강 파이프라인 문서를 공유하여 팀 내 공통 기준을 마련함

## 내일 목표 / 할 일

- SpeechT5를 VITS로 교체한 데모 구현
